# Septic Station — Agent Instructions

This directory manages a DuckPond instance running on a BeaglePlay
(`septicplaystation.local`) that collects and visualizes Modbus/BME280
telemetry from an Orenco septic system.

## Architecture

- **BeaglePlay** runs `podman` with the `duckpond:latest-arm64` container image.
- An OTel Collector on the device writes OTelJSON logfiles to `/home/data/`.
- `pond.sh` wraps every `pond` CLI call via SSH + podman on the remote host.
- Config files (YAML, markdown templates) live in this directory and are
  `scp`'d to the remote at `/home/debian/config/`.

## Scripts (run from this directory)

| Script | Purpose | When to use |
|--------|---------|-------------|
| `build.sh` | Cross-compile arm64 container image, push to BeaglePlay | After code changes to duckpond |
| `setup.sh` | Wipe pond, init, create dirs, copy templates, install factories | First time or full reset |
| `update.sh` | Push updated configs, recreate factories with `--overwrite` | After editing YAML or site templates |
| `run.sh` | Ingest new logfile data (`pond run /etc/ingest`) | To pull in new telemetry data |
| `generate.sh` | Run sitegen, rsync built site to `export/` | To build the static site |

## Full Reset & Site Generation (the standard workflow)

```bash
# 1. Build and push image (only if duckpond code changed)
./build.sh

# 2. Initialize the pond from scratch
./setup.sh

# 3. Ingest logfile data
./run.sh

# 4. Generate the static site (lands in export/)
./generate.sh

# 5. Preview locally
npx vite export --port 4174 --open
```

If the pond already exists and you only changed configs or templates:

```bash
./update.sh       # push configs + recreate factories
./run.sh          # re-ingest if needed
./generate.sh     # rebuild site
```

## Key Details

- `setup.sh` is destructive — it wipes the podman volume. Only use for a
  full reset.
- `pond copy host:///config/site /etc/site` requires the parent `/etc` to
  exist in the pond. `setup.sh` creates it via `mkdir -p /etc`.
- The ingest factory reads from `/data` (host mount) and writes to
  `/ingest/` in the pond.
- Temporal-reduce (`/reduced`) is a dynamic factory — computed on read,
  no explicit run step needed.
- Sitegen exports to `/output` inside the container, which is mounted at
  `/home/debian/site-output` on the host, then rsynced to `export/`.
- The generated site includes 6 metric groups (pumps, cycle-times,
  pump-modes, flow-totals, dose-zones, environment) at 3 resolutions
  (1h, 6h, 1d).

## Troubleshooting

- **"Not a Delta table"** on `pond list` → pond isn't initialized; run `setup.sh`.
- **"NotFound: /etc/site"** during setup → `mkdir -p /etc` is missing before
  the `pond copy` step.
- **No rotated logfiles** → only the active `septicstation.json` exists; that's
  fine, the ingest factory handles it.
- **SSH banners in output** → normal; the BeaglePlay prints its MOTD on every
  SSH connection. Ignore them.
- **Large allocation warnings** → sitegen allocates ~64MB per series export.
  Watch total memory on the BeaglePlay (1GB RAM).
